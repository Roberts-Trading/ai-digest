# AI Digest — 2026-05-27

## Product Updates

### Anthropic — Free Security Plugin for Claude Code, 28 Enterprise Integrations
- **Summary**: Two major releases. (1) Anthropic shipped a free security-guidance plugin for Claude Code that reviews code edits, model outputs, and commits in real time. Runs lightweight pattern checks without model calls (zero cost), deeper reviews via Opus 4.7 on commits/pushes. Internal testing showed 30-40% decrease in security-related PR comments. Open-source reference repo: `anthropics/claude-code-security-review`. (2) Claude Compliance API launched with 28 SIEM/DLP integrations (CrowdStrike, Netskope, etc.) — enterprise teams can now route Claude conversation content and activity logs into their existing security dashboards.
- **Source**: https://www.helpnetsecurity.com/2026/05/27/anthropic-claude-code-security-guidance-plugin/ ; https://www.techtimes.com/articles/317272/20260527/claude-enterprise-security-integrations-28-vendors-now-route-ai-activity-existing-siem-dlp-tools.htm
- **Significance**: The security plugin is the first free, built-in security layer for any coding agent. It shifts vulnerability detection into the development session itself, not just at PR review. The Compliance API is the enterprise trust play — if you can't monitor it, you can't approve it. Both moves position Anthropic as the enterprise-safe AI choice.

### Google — Open-Sources Agent Executor + Agent Substrate
- **Summary**: Google open-sourced two production agent infrastructure tools. Agent Executor is a runtime standard for long-running AI agents — handles durable execution, checkpointing, connection recovery, and trajectory branching. Agent Substrate is a Kubernetes-native orchestration layer for hyperscale agent workloads. Both integrate with LangChain, LangGraph, Gemini API, and Google's Agent Development Kit.
- **Source**: https://www.opensourceforu.com/2026/05/google-open-sources-agent-executor-for-production-ai-agents/
- **Significance**: This is Google's answer to the "how do agents run in production?" problem. Agent Executor's checkpointing and durable execution directly competes with LiteLLM Agent Platform. The trajectory branching feature (testing multiple paths from checkpoints) is novel — it's version control for agent workflows.

### OpenAI — Codex Hooks GA, Conversational Ads Data
- **Summary**: Codex Hooks went GA on May 14 — a control layer for governing AI agent behavior in production. Meanwhile, Similarweb data shows ChatGPT ads are building a new revenue engine around conversational intent, not keywords. 46% of users who saw ads started with no commercial intent — the conversation itself creates the ad opportunity. Cursor is among the brands buying ChatGPT ads.
- **Source**: https://byteiota.com/openai-codex-hooks-control-coding-agent/ ; https://www.businessinsider.com/chatgpt-ads-big-threat-google-openai-similarweb-search-keywords-conversations-2026-5
- **Significance**: Conversational intent is a fundamentally different ad model from keyword search. If it works, it threatens Google's core revenue. The fact that 46% of ad impressions came from users with no initial commercial intent is the key number — it means ChatGPT can manufacture ad inventory from any conversation.

### JetBrains — Koog 1.0 Stable
- **Summary**: JetBrains shipped Koog 1.0, its open-source framework for building AI agents in Kotlin and Java. Stable core with guaranteed no breaking changes for one year. Features: local Android AI (LiteRT models), OpenTelemetry support, Anthropic prompt caching, improved persistence and memory for long-running agents.
- **Source**: https://blog.jetbrains.com/ai/2026/05/koog-1-0-is-out-stable-core-better-interop-and-multiplatform-observability/
- **Significance**: First production-stable agent framework for the JVM ecosystem. The Anthropic prompt caching and OpenTelemetry support suggests JetBrains is building for enterprise teams already running Kotlin/Java stacks — a different entry point than the Python-native agent frameworks.

### Cognition — $1B Raise at $26B Valuation
- **Summary**: Cognition (Devin) raised $1B at $26B valuation. CEO argues value is in the agent layer (orchestration + execution), not the model layer. Uses OpenAI and Anthropic models — the same companies building competing coding products. Explicitly positioning for independence vs. the SpaceX-Cursor consolidation.
- **Source**: https://thenextweb.com/news/cognition-just-raised-1-billion-at-a-26-billion-valuation-and-90-of-its-own-code-is-written-by-its-ai
- **Significance**: The bet that the model layer commoditizes while the agent layer captures durable value is the core contention in AI tooling right now. If it's right, Cognition is undervalued. If it's wrong, they're a thin wrapper on commoditized APIs.

## People & Commentary

### Aaron Levie — "AI Psychosis"
- **What they said/did**: Box founder Aaron Levie wrote that "CEOs are uniquely prone to AI psychosis because they're sufficiently distant from the last mile of work that still has to happen." TechCrunch reports 115,430 layoffs from 152 companies in the first 5 months of 2026 — nearly matching all of 2025. Many companies are "AI washing" their cuts.
- **Source**: https://techcrunch.com/2026/05/27/tech-ceos-are-apparently-suffering-from-ai-psychosis/
- **Context**: The gap between CEO expectations and agent reality is widening. MIT researchers found agents still aren't doing human-quality work — they project base competence on most text tasks by 2029. NBER research shows perceived productivity gains are larger than measured gains.

### Thorsten Meyer — Enterprise Revenue Lock as Valuation Argument
- **What they said/did**: Meyer argues both OpenAI and Anthropic are racing to convert enterprise-revenue lock into their load-bearing valuation argument before public markets demand audited proof. The agents producing the enterprise revenue are the same agents whose disruption funds the multiple that funds the compute that builds the agents. A $1T valuation against a $25B run rate is ~40x revenue.
- **Source**: https://thorstenmeyerai.com/reality-check/the-runway-how-enterprise-revenuelock-becomes-the-load-bearing-valuation-argument/
- **Context**: This is the sharpest structural analysis of the AI IPO window. The reflexivity is the risk — if enterprise revenue disappoints, the multiple collapses, compute funding tightens, and the agents that produce the revenue get worse. It's a feedback loop that runs in both directions.

### Sunil Sharan (Fortune) — Altman Fooled Pichai
- **What they said/did**: Argues Sam Altman has built a market consensus around AI as an intelligent mind, when it's actually an expensive pattern-matching system prone to errors. Google's response — cannibalizing Search with AI — may prove catastrophically self-defeating.
- **Source**: https://fortune.com/2026/05/27/sam-altman-fooled-sundar-pichai-google-ai-search-bust-sunil-sharan/
- **Context**: Provocative take, but the core question is real: did Google have to rebuild Search around AI, or did Altman's framing force a premature response?
