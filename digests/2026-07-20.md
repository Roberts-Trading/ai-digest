# AI Digest — 2026-07-20

## Product Updates

### Anthropic — Fable 5 Billing Split Takes Effect
- **Summary**: At midnight Pacific July 20, the permanent two-tier Fable 5 access structure went live. Max and Team Premium: Fable 5 bundled at 50% of standard weekly usage limits, no per-token charge. Pro and Team Standard: $100 one-time credit, then $10/$50 per 1M in/out tokens (Anthropic's steepest GA rate). The 50% Claude Code weekly rate-limit boost (in place since May 13) also ended today — so standard limits shrink first, then the 50% Fable cap applies against the reduced baseline. Power users will feel the compression. A single intensive session generating 2M output tokens exhausts the entire $100 credit. Max 5x ($110/mo), Max 20x ($220/mo), Team Premium ($100/mo). The "Fable Time" era of rushes and extensions is over — access is now permanent but stratified.
- **Source**: https://www.techtimes.com/articles/320999/20260720/claude-fable-5-billing-splits-today-max-gets-it-free-pro-pays-per-token.htm ; https://www.androidauthority.com/claude-fable-5-access-3689068/
- **Significance**: The two-tier reality is now permanent. Frontier models are a premium feature, not a baseline. The compound effect (50% boost ending + 50% Fable cap on reduced baseline) means even Max subscribers see meaningful reduction from the promotional period. The $100 credit for Pro users is a stopgap — a single intensive session can exhaust it. This is the new normal until compute capacity catches up.

### Letta — Memory Architecture Comparison and Positioning
- **Summary**: DEV Community article compares memory tools on editability and auditability. Letta positioned as "agent state blocks" — stored in database, accessible via API and visualizer, but not readable as flat files. Comparison table: Mem0 (vector store, no erasure proof), Letta (agent state blocks, no erasure proof), Zep/Graphiti (temporal knowledge graph, partial erasure), Cognee (graph+vector+relational), ChatGPT memory (opaque, no audit log), Claude Code CLAUDE.md (markdown file, git diff = erasure proof), PLUR (YAML files, git diff = erasure proof). Key finding: "file-based memory is auditable; database-based memory is not." Recommendation: "Self-managing stateful agent with memory tiers → Letta." Also: "Building My Memory Layer" article describes a 7-layer architecture (input understanding, decision engine, memory network, active/non-active memory, retrieval engine, context validation, language model) — each layer emerging from solving a failure. Rillet's Aura Memory launches as a permanent memory layer for AI-native ERP with full audit trail.
- **Source**: https://dev.to/plur9/how-do-i-make-my-ai-agents-memory-editable-and-auditable-37aa ; https://dev.to/venu_varma/building-my-memory-layer-the-thinking-process-behind-every-layer-1apb ; https://www.rillet.com/blog/aura-memory-ai-accounting-automation
- **Significance**: The auditability gap between file-based and database-based memory is the key architectural tension. Letta's state-block approach provides API-driven management but can't prove erasure (state blocks in database). The PLUR/YAML approach (git diff = erasure proof) is the auditability gold standard. The 7-layer "Building My Memory Layer" architecture (especially the Context Validation Layer — "is this memory still true today?") aligns with the bitemporal memory concept from yesterday. Aura Memory's "live data wins over remembered instructions" principle is the right pattern for enterprise memory.

## New Tools & Discoveries

### Grok Build (Open Source)
- **What it does**: Terminal-based coding agent, now Apache 2.0. Rust workspace, 1M+ lines. Agent loop, tools, terminal UI, extension system (skills, plugins, hooks, MCP servers, subagents). Compile locally with self-hosted inference. Windows support now available.
- **Why notable**: Real-world reference for agent loop architecture, terminal UX, tool dispatch, sandboxing, memory, extensibility. But: no external PRs accepted, security via HackerOne. Source-transparency, not community-governed.

### Cartha (Agent Fleet Ops)
- **What it does**: Ops layer for AI agent fleets in production. Server-enforced memory scopes (user/agent/team/org), fail-closed spend budgets, nested agents with attenuated delegation, policies and escalations, dashboard for agents/traces/cost/memory. Python SDK wraps OpenAI.
- **Why notable**: Addresses the governance gap — "observability tells you what happened, governance decides what is allowed to happen next." Scoped memory without leaks, hard budgets that stop retry loops, tool limits for child agents. The right architecture for production agent fleets.

### Aura Memory (Rillet)
- **What it does**: Permanent memory layer for AI-native ERP. Tell it a formatting rule once, applies in all future sessions. States which memory it used in each response. Full audit trail. Live GL data overrides remembered instructions when they conflict.
- **Why notable**: "Agents propose; you approve. Full audit trail." The live-data-wins-over-memory principle is the right pattern for enterprise — memory is a hint, not a source of truth. The "states which memory it used" transparency is what's missing from most memory systems.
