# AI Digest — 2026-05-31

## Product Updates

### NVIDIA — Open Source Physical AI Agent Toolkit
- **Summary**: NVIDIA released a major open source collection of physical AI agent skills and tools spanning Omniverse, Cosmos, Alpamayo, Metropolis, Isaac, and Jetson. Turns complex physical AI workflows (robotics, AV, vision AI, digital twins) into agent-executable tasks. Available on GitHub and skills.sh. Also launched Alpamayo 2 Super Open Reasoning Model for robotaxis.
- **Source**: https://nvidianews.nvidia.com/news/nvidia-releases-major-collection-of-open-source-agent-tools-and-skills-for-physical-ai
- **Significance**: NVIDIA is going all-in on the skills standard. The fact that their physical AI skills are available on skills.sh (the same format Claude Code, Cursor, Codex, Letta use) confirms the "skills as packages" convergence. Physical AI is the next agent frontier — robots, autonomous vehicles, and industrial systems.

### Microsoft Foundry — May 2026 Updates
- **Summary**: Foundry Agent Service now supports skills and toolboxes (no Microsoft Agent Framework required). SDK 2.2.0 adds external agent definitions, model weight registry, skills versioning, optimization jobs, routines, and memory stores. Foundry Local 1.1/1.2 adds live audio transcription, Qwen 3.5 Vision, multilingual ASR. Evaluation infrastructure now grades real traces from agents running anywhere.
- **Source**: https://devblogs.microsoft.com/foundry/whats-new-in-microsoft-foundry-may-2026/
- **Significance**: Skills are the common thread again — Microsoft is adopting the same skill/toolbox pattern. The model weight registry is the enterprise piece: programmatic model management that previously required the portal. The evaluation infrastructure upgrade (grading real traces, not just synthetic tests) is the most practical improvement.

### AI Factory — One-Command AI Dev Pipeline
- **Summary**: New npm package that auto-detects your project stack, installs relevant skills and MCP services, and sets up a structured plan→implement→verify→fix workflow. Spec-driven development: AI must create a plan before writing code. Every step pausable, resumable, reviewable.
- **Source**: https://pythonlibraries.substack.com/p/one-command-to-set-up-your-ai-dev
- **Significance**: The "plan-first" philosophy aligns with Anthropic's dynamic workflows. The zero-config setup addresses a real pain point — every new project needs 30 minutes of skill/MCP/CI configuration before you can start. If this works, it becomes the Docker Compose of AI dev environments.

### Claude Mythos — Security Vulnerability Discovery
- **Summary**: VentureBeat deep dive on Mythos's security capabilities. Mythos scored 83.1% on CyberGym vulnerability reproduction benchmark. One OpenBSD campaign across 1,000 scaffold runs cost under $20,000 in compute. Exploitation timelines collapsing: CVE-2026-33017 exploited in 20 hours, CVE-2026-39987 in 9 hours 41 minutes. Project Glasswing has already uncovered 10,000+ high/critical-severity vulnerabilities.
- **Source**: https://venturebeat.com/security/claude-mythos-exposed-a-hard-truth-your-enterprise-patching-process-is-way-too-slow
- **Significance**: The security implications are real. Mythos can find zero-days at $20K per campaign. Exploit windows are now measured in hours, not days. The bottleneck is no longer finding vulnerabilities — it's the human workload to verify, disclose, and patch. This is why Anthropic is holding Mythos back: the defensive infrastructure to handle what it finds doesn't exist yet.
