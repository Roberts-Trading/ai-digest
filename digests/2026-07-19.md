### Letta — Memory Architecture Discourse Deepens
- **Summary**: Multiple articles advance the memory architecture conversation: (1) **Bitemporal AI memory** — tracking two clocks (event time: when fact was valid; system time: when AI learned it). Enables three queries: current truth, what was known at decision time, revised historical truth. Critical for audits, reproducible evaluations, incident reconstruction. (2) **Production memory at scale** — treating memory as first-class system component with distinct stores (working, episodic, semantic). Multi-signal retrieval (vector similarity + recency + importance tagging). Asynchronous writes to avoid blocking inference. (3) **7-layer ZenBrain architecture** — HiMeS with sleep consolidation (replays recent experiences during idle periods, strengthens important memories, prunes weak ones via Hebbian learning). FSRS spaced repetition for long-term retention. Open source, Apache 2.0. (4) **Memory-safe AI development specs** — framework for using coding agents to modify systems with persistent memory. "Don't delete memories by default — lower importance score, deactivate, or archive." Distilled from 11 deployed AI assistants across 8 model architectures.
- **Source**: https://dev.to/ethanbeirne/bitemporal-ai-memory-how-to-preserve-what-an-agent-knew-then-2g96 ; https://programminginsider.com/ai-agent-memory-in-production-what-breaks-at-scale-and-how-teams-are-solving-it/ ; https://dev.to/goldfinger2025/7-layer-memory-architecture-how-zenbrain-remembers-like-a-human-brain-jb1
- **Significance**: The memory architecture field is maturing fast. Bitemporal memory is the most important new concept — it solves the "what did the agent know and when" problem that plain RAG cannot. The sleep consolidation pattern (also seen in Google's Always-On Memory Agent from yesterday) is becoming a standard approach. The "don't delete memories, archive them" principle aligns with how Letta's own memory system works. The field is moving from "how to store" to "how to govern memory lifecycle."

## New Tools & Discoveries

### Bitemporal AI Memory (Lians)
- **What it does**: Open-source bitemporal memory and record layer for AI agents. Tracks event time and system time independently. Supports current truth, known-at-decision-time, and revised-historical-truth queries. Immutable version identities, provenance, deterministic as-of queries.
- **Why notable**: Solves the "what did the agent know when it made this decision" problem. Critical for compliance, audits, incident reconstruction, and reproducible evaluations. Plain RAG cannot do this.

### ZenBrain HiMeS (7-Layer Memory)
- **What it does**: Open-source (Apache 2.0) hierarchical memory system with sleep consolidation, Hebbian knowledge graphs, and FSRS spaced repetition. Pure TypeScript, zero dependencies. 276 tests. 9 foundational algorithms.
- **Why notable**: The sleep consolidation pattern (replay, strengthen, prune during idle) is the same approach Google's Always-On Memory Agent uses. The FSRS integration for long-term retention is novel — borrowing from flashcard scheduling to optimize when memories should be reviewed.
